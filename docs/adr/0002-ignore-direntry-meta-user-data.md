# ADR 0002: DirEntry의 `meta.user_data`는 도메인 모델에서 사용하지 않는다

- 상태: 승인됨
- 날짜: 2026-03-24

## 배경

`radosgw-admin bi list`의 `DirEntry.meta`에는 `user_data` 필드가 포함될 수 있다.

하지만 이 필드는 Swift 호환 시나리오를 위한 값이며, 현재 이 프로젝트가 다루는 대상과 직접 관련이 없다.

현재 프로젝트의 관심사는 Swift 전용 메타데이터를 보존하거나 해석하는 것이 아니라, BI 엔트리를 필요한 수준으로 해석하는 것이다.

## 결정

`DirEntry.meta.user_data`는 `internal/pkg/domain`의 도메인 모델에 포함하지 않는다.

즉, `Meta`에는 `userData` 필드를 두지 않으며, 파싱 입력에 해당 값이 존재하더라도 도메인 의미로 승격하지 않는다.

## 근거

- `user_data`는 Swift 전용 필드다.
- 현재 프로젝트의 요구사항은 Swift 지원이 아니다.
- 사용하지 않는 필드를 도메인 모델에 유지하면 의미 없는 복잡도만 증가한다.
- 도메인 모델은 실제로 해석하고 사용하는 정보만 담는 편이 더 명확하다.

## 결과

긍정적 영향은 아래와 같다.

- `Meta` 구조가 단순해진다.
- 도메인 모델이 현재 요구사항에 더 정확하게 맞춰진다.
- Swift 전용 데이터가 핵심 모델에 불필요하게 스며드는 것을 막을 수 있다.

주의사항은 아래와 같다.

- 향후 Swift 관련 요구사항이 추가되면 이 결정을 재검토해야 한다.
- 원본 JSON에는 `user_data`가 남아 있을 수 있으나, 현재 도메인 계층에서는 무시한다.
